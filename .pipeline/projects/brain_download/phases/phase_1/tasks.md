# Phase 1 Tasks

- [ ] Task 1: Core Deconstruction, Selection & Sequencing Engine
  - What: Build the foundational meta-learning engine that deconstructs any topic into a skill tree, applies Pareto-optimal (80/20) selection, and produces an optimally sequenced course outline using the DESSC framework.
  - Files:
    - `brain_download/core/__init__.py`
    - `brain_download/core/deconstruction_engine.py` — Topic breakdown into skill components; uses LLM prompt templates to generate a hierarchical skill tree with importance scores
    - `brain_download/core/selection_matrix.py` — Pareto filtering logic; identifies the 20% of skills that deliver 80% of outcomes; configurable threshold
    - `brain_download/core/sequencing_engine.py` — Prerequisite-aware ordering algorithm; produces a linear learning path from the skill tree with module groupings
    - `brain_download/core/models.py` — Dataclasses for Topic, SkillNode, SkillTree, CourseModule, CourseOutline
    - `brain_download/config/learning_models.py` — DESSC framework parameters, default thresholds, and meta-learning model configs
    - `brain_download/config/domain_profiles.py` — Subject-specific profiles (e.g., coding, language, business) with domain-specific deconstruction hints
  - Done when:
    - Given a topic string (e.g., "Python for Data Science"), the deconstruction engine returns a SkillTree with at least 15 nodes across 3+ levels of hierarchy
    - The selection matrix correctly identifies a Pareto-optimal subset (configurable ~20% of nodes)
    - The sequencing engine produces a linear CourseOutline with modules ordered by prerequisite dependencies
    - Domain profiles can override deconstruction hints (e.g., "Python for Data Science" emphasizes numpy/pandas over web frameworks)
    - All models are serializable to/from JSON

- [ ] Task 2: Stakes & Compression Engine
  - What: Build the accountability mechanism designer and information density optimizer that works on top of the sequenced course outline.
  - Files:
    - `brain_download/core/stakes_calculator.py` — Generates accountability suggestions per module (deadlines, peer review, public commitments, financial stakes); scored by motivational impact
    - `brain_download/core/compression_engine.py` — Identifies what to skip, what to emphasize, and what to compress per module; produces a compression map with density targets
    - `brain_download/core/__init__.py` — Updated to export StakesConfig and CompressionMap
  - Done when:
    - Given a CourseOutline, stakes_calculator produces at least 3 accountability mechanisms per module with impact scores
    - compression_engine produces a per-module compression map with: skip_list, emphasize_list, compress_ratio, and estimated_time_savings
    - Both engines produce JSON-serializable outputs
    - Stakes mechanisms are categorized by type (social, financial, temporal, identity-based)

- [ ] Task 3: Video Script Generator with Tone Engine
  - What: Build the script generator that produces per-lesson video scripts with hooks, core content, examples, recaps, and CTAs. Include a tone engine for voice customization and a template library for consistency.
  - Files:
    - `brain_download/scripts/__init__.py`
    - `brain_download/scripts/video_script_generator.py` — Takes a CourseOutline and generates a VideoScript per module; each script contains: title, estimated_duration_minutes, hook, core_content, examples (2-3), recap, call_to_action, section_breaks
    - `brain_download/scripts/tone_engine.py` — Tone customization system; supports at least 4 tones (conversational, authoritative, humorous, inspirational); modifies script language, pacing, and example style per tone
    - `brain_download/scripts/script_templates.py` — Structured prompt templates for LLM-based script generation; template variables for topic, module, tone, duration
    - `brain_download/scripts/models.py` — Dataclasses for VideoScript, ScriptSection, ToneConfig
  - Done when:
    - Given a CourseOutline, the generator produces a VideoScript for each module
    - Each script contains all required sections: hook, core_content, examples, recap, CTA
    - Estimated duration is reasonable (5-15 min per script, total course 60-120 min)
    - At least 4 tone options produce distinctly different scripts (verified by comparing language patterns)
    - Scripts are Udemy-ready: proper pacing, section breaks, clear learning objectives per video
    - All outputs are JSON-serializable

- [ ] Task 4: CLI Interface
  - What: Build the command-line interface that ties the entire pipeline together. Users should be able to generate a complete course with a single command.
  - Files:
    - `brain_download/__init__.py`
    - `brain_download/__main__.py`
    - `brain_download/cli.py` — CLI with click/typer; command: `brain-download create --topic "..." --output ./course/ [--tone conversational] [--format json|markdown]`
    - `brain_download/pipeline.py` — Orchestrator that chains: deconstruction → selection → sequencing → stakes/compression → script generation → output
    - `brain_download/output_formatters.py` — Converts course data to JSON and Markdown formats for output
  - Done when:
    - `brain-download create --topic "Python for Data Science" --output ./course/` runs end-to-end without errors
    - Output directory contains: course_outline.json, course_outline.md, and per-module script files
    - `--tone` flag works and produces tone-appropriate scripts
    - `--format` flag supports both json and markdown output
    - CLI shows progress feedback during generation
    - Help text (`--help`) documents all options

- [ ] Task 5: Tests & End-to-End Verification

  - Files:

    - `brain_download/tests/test_deconstruction.py` — Tests skill tree generation, Pareto selection, domain profile overrides
    - `brain_download/tests/test_sequencing.py` — Tests prerequisite ordering, module grouping
    - `brain_download/tests/test_stakes.py` — Tests accountability mechanism generation
    - `brain_download/tests/test_compression.py` — Tests compression map generation
    - `brain_download/tests/test_scripts.py` — Tests script generation, tone differences, required sections

    - `brain_download/tests/conftest.py` — Shared fixtures and test data
    - `brain_download/tests/test_integration.py` — Full pipeline test: topic → course outline + scripts

    - `pytest` runs clean with no warnings
