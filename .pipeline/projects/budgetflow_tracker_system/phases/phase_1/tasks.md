# Phase 1 Tasks

- [ ] Task 1: Project scaffolding and core data models
  - What: Create the BudgetFlow project structure with pyproject.toml, package layout, Pydantic data models, and configuration management. Set up the SQLite connection layer (WAL mode, connection pooling) and define the full schema (accounts, transactions, categories, budgets, transaction_rules) as migration scripts.
  - Files: budgetflow_tracker/pyproject.toml, budgetflow_tracker/src/core/__init__.py, budgetflow_tracker/src/core/models.py (Pydantic models), budgetflow_tracker/src/core/database.py (SQLite engine + migration system), budgetflow_tracker/src/core/config.py (settings), budgetflow_tracker/data/default_rules.db (seeded with default categories and rules), budgetflow_tracker/config/budgetflow.yaml (default config)
  - Done when: pyproject.toml installs the package with `pip install -e .`; `budgetflow_tracker.db` is created on first import with all five tables and proper indexes; Pydantic models validate correctly; default categories (Food & Drink, Transportation, Shopping, Entertainment, Bills & Utilities, Healthcare, Income, Other) and 20+ default categorization rules are seeded; no network calls are made during initialization.

- [ ] Task 2: Multi-bank CSV import pipeline
  - What: Build a CSV parser that auto-detects bank format (Chase, Wells Fargo, Bank of America, Capital One) by inspecting header patterns, maps columns to a canonical schema (date, description, amount, type), validates each row, and bulk-inserts into SQLite. Include a manual column-mapping fallback for unrecognized formats.
  - Files: budgetflow_tracker/src/import/__init__.py, budgetflow_tracker/src/import/csv_parser.py (format detection, column mapping, validation, bulk insert), budgetflow_tracker/tests/test_csv_parser.py (unit tests with synthetic CSVs from each bank)
  - Done when: Parser correctly imports CSVs from Chase, Wells Fargo, Bank of America, and Capital One with zero manual configuration; unrecognized formats trigger a column-mapping prompt; all imported rows pass validation (date parseable, amount numeric, non-null description); bulk insert completes in <2 seconds for 10,000 rows; all tests pass.

- [ ] Task 3: Rule-based categorization engine
  - What: Implement a deterministic rule engine that applies user-defined and default categorization rules to transactions. Rules use pattern matching (substring, regex, exact match) against merchant/description fields. Engine supports rule priority ordering, conflict resolution (first-match-wins), and a confidence score per categorization. Include a default rule set covering common merchants and keywords.
  - Files: budgetflow_tracker/src/categorize/__init__.py, budgetflow_tracker/src/categorize/rule_engine.py (rule application, priority ordering, confidence scoring), budgetflow_tracker/src/categorize/default_rules.py (comprehensive default rule set), budgetflow_tracker/tests/test_rule_engine.py (tests for pattern matching, priority, confidence, edge cases)
  - Done when: Engine correctly categorizes ≥80% of a representative test dataset (500+ transactions from multiple banks) on first pass using only default rules; rules can be added/edited/deleted via database; confidence scores are computed and stored per transaction; all tests pass including edge cases (empty descriptions, special characters, case-insensitive matching).

- [ ] Task 4: Budget management (CLI)
  - What: Build CLI commands for creating, editing, and viewing budgets per category and time period (monthly/weekly). Store budgets in the `budgets` table. Implement spend-vs-budget progress calculation with percentage and remaining amounts.
  - Files: budgetflow_tracker/src/ui/cli_budget.py (CLI commands: create, edit, list, delete budgets), budgetflow_tracker/src/budget/__init__.py, budgetflow_tracker/src/budget/engine.py (budget CRUD + spend calculation), budgetflow_tracker/tests/test_budget.py (tests for budget CRUD, spend calculation, rollover)
  - Done when: User can create a monthly budget for at least 3 categories via CLI; budgets persist in SQLite and survive restarts; spend-vs-budget progress is calculated correctly for any date range; rollover logic works (unused budget rolls to next period); all tests pass.

- [ ] Task 5: CLI dashboard — spending summaries and budget progress
  - What: Build the terminal-based dashboard that displays (a) spending by category over a configurable date range, (b) budget progress bars for active budgets, and (c) a summary table of top spending categories. Support date range filtering via CLI arguments. Use rich library for terminal formatting (progress bars, tables, colors).
  - Files: budgetflow_tracker/src/ui/cli_dashboard.py (main CLI dashboard with rich formatting), budgetflow_tracker/src/ui/cli.py (CLI entry point with argparse subcommands: import, categorize, budgets, dashboard), budgetflow_tracker/src/reports/summary.py (spending aggregation queries), budgetflow_tracker/tests/test_dashboard.py (tests for date range filtering, category aggregation, progress bar rendering)
  - Done when: Dashboard renders a formatted table of spending by category for any date range; budget progress bars display correctly (e.g., "Food & Drink: ████████░░ 80% — $200 of $250"); top spending categories are highlighted; date range filtering works via CLI flags (e.g., `--from 2024-01-01 --to 2024-01-31`); all tests pass; end-to-end flow works: import CSV → auto-categorize → view dashboard → set budgets → view progress.
