  - What: Build the IoT data ingestion layer that connects real-world sensor data and tracking inputs into the Chronovision world model.   - Files:

    - chronovision/src/iot/protocol_opcua.py (OPC-UA protocol adapter)

    - chronovision/src/iot/streaming.py (Kafka/Pulsar streaming connector)
    - chronovision/src/iot/sensor_registry.py (sensor discovery and registration)
    - chronovision/src/iot/updater.py (IoT event handler that updates StateSpace)
    - chronovision/tests/test_iot.py
  - Done when:
    - MQTT broker can connect, subscribe to topics, and publish sensor events
    - OPC-UA protocol adapter can read sensor values from a mock OPC-UA server

    - IoT updater correctly updates StateSpace entities with sensor data
    - All tests pass with pytest

    - chronovision/src/predictor/horizon_predictor.py (main multi-horizon predictor combining all components)
