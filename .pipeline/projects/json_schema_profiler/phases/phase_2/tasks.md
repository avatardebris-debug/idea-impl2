# Phase 2 Tasks

- [x] Task 1: Anomaly detection module
  - What: Create `src/json_schema_profiler/anomaly.py` with functions to detect per-field anomalies: null rates (configurable threshold), numeric outliers (IQR method), cardinality anomalies (high-cardinality strings flagged), and numeric skew detection. Each anomaly must include a confidence score (0.0–1.0) and a suggested fix. Return a dict keyed by field name with per-field anomaly lists.
  - Files: `src/json_schema_profiler/anomaly.py` (new), `src/json_schema_profiler/__init__.py` (export new symbols)
  - Done when: (a) `detect_anomalies(data, thresholds={...})` returns anomaly dict with per-field anomalies; (b) null rate anomalies trigger when null_pct > threshold (default 0.10); (c) IQR-based numeric outliers are detected and scored; (d) cardinality anomalies flag string fields with >50 unique values; (e) skew detection flags numeric fields with |skew| > 2; (f) each anomaly has `confidence` (float 0–1) and `suggested_fix` (str); (g) ≥10 unit tests covering each anomaly type, edge cases (empty data, all-null fields, single-value fields), and threshold customization.

- [x] Task 2: Parquet file reader and schema inference
  - What: Create `src/json_schema_profiler/parquet_loader.py` with functions to read `.parquet` files using pyarrow. Support single file and directory scanning. Infer schema from both pyarrow's native schema metadata and sampled data stats (null counts, value distributions). Merge pyarrow schema with data-driven stats.
  - Files: `src/json_schema_profiler/parquet_loader.py` (new), `src/json_schema_profiler/__init__.py` (export new symbols)
  - Done when: (a) `read_parquet(path)` returns a list of dicts matching the pyarrow schema; (b) `infer_parquet_schema(path)` returns a schema dict that matches pyarrow's native types for all columns; (c) directory scanning via `scan_parquet_directory(dir_path)` processes all `.parquet` files and merges schemas; (d) handles empty parquet files gracefully; (e) ≥5 unit tests with sample parquet files created programmatically (using pyarrow + pandas).

- [x] Task 3: Output formatters (YAML and Pydantic)
  - What: Create `src/json_schema_profiler/formatters.py` with functions to produce three output formats: (a) `jsonschema` — enhanced JSON Schema with anomaly metadata embedded as custom `x-anomalies` extension; (b) `yaml` — structured YAML with schema, anomaly report, confidence scores, and suggested fixes; (c) `pydantic` — Python code that generates a Pydantic BaseModel class with field validators derived from inferred constraints.
  - Files: `src/json_schema_profiler/formatters.py` (new), `src/json_schema_profiler/__init__.py` (export new symbols)
  - Done when: (a) `to_jsonschema(schema, anomalies)` produces valid JSON Schema draft-07 with `x-anomalies` per field; (b) `to_yaml(schema, anomalies)` produces valid, parseable YAML with schema, anomalies, confidence, and fixes; (c) `to_pydantic(schema, anomalies)` produces valid Python source code that can be exec'd to define a Pydantic model; (d) all three outputs are deterministic and idempotent; (e) ≥6 tests verifying each format's output structure and validity.

- [x] Task 4: CLI `analyze` command with directory scanning and format selection
  - What: Extend `cli.py` to add an `analyze` subcommand that is a superset of `infer`. It must: (a) accept `.json` and `.parquet` files; (b) accept directory paths and scan for all `.json` / `.parquet` files; (c) support `--format jsonschema | yaml | pydantic`; (d) support `--thresholds` for anomaly detection configuration (e.g., `--thresholds null_rate=0.15 iqr_multiplier=1.5`); (e) embed anomaly report in output; (f) remain backwards compatible — `infer` command still works as-is.
  - Files: `src/json_schema_profiler/cli.py` (modify), `src/json_schema_profiler/__init__.py` (if needed)
  - Done when: (a) `json-schema-profiler analyze tests/fixtures/simple.json --format yaml` outputs valid YAML with schema + anomalies; (b) `json-schema-profiler analyze tests/fixtures/` scans all files in directory; (c) `json-schema-profiler analyze data.parquet` reads parquet and outputs schema with anomalies; (d) `--thresholds` option works and affects anomaly detection; (e) `json-schema-profiler infer` still works unchanged (backwards compatible); (f) `--format pydantic` outputs valid Python code; (g) directory scanning processes mixed `.json` and `.parquet` files in one run.
