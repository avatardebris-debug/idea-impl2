- [ ] Task 2: Clause library (JSON-backed registry with 20+ clauses)
  - What: Build core/clause_library.py — a JSON-backed clause registry that loads clause definitions from a JSON file. Each clause has: name, default value, allowed values, description, and jurisdiction overrides. Create the JSON data file with 20+ clauses covering: definition_of_confidential_info, exclusions_from_confidential_info, term_length, permitted_use, return_of_materials, remedies, governing_law, dispute_resolution, non_solicitation, non_circumvention, no_license_granted, survival_period, notice_period, confidentiality_obligations, data_protection, third_party_disclosure, injunctive_relief, liquidated_damages, mutual_vs_unilateral, carve_outs, standard_of_care, publication_restrictions, agent_disclosure, audit_rights.
  - Files: core/clause_library.py, core/clause_data.json, tests/test_clause_library.py
  - Done when: clause_library.py exposes ClauseLibrary class with methods: get_clause(name), get_all_clauses(), get_allowed_values(name), apply_overrides(jurisdiction), override_clause(name, value); clause_data.json contains 20+ clauses with defaults, allowed values, and descriptions; tests verify loading, retrieval, override, and jurisdiction override functionality (≥ 8 test cases)

- [ ] Task 3: Jurisdiction database with 3 jurisdictions
  - What: Build core/jurisdiction_db.py and the three jurisdiction modules. Each jurisdiction defines: name, display_name, governing_law, default_term, required_clauses, optional_clauses, default_clause_values, and any special notes. The registry supports lookup by jurisdiction key and returns merged config (base defaults + jurisdiction overrides).
  - Files: core/jurisdiction_db.py, jurisdictions/base.py, jurisdictions/us/california.py, jurisdictions/uk/england_wales.py, jurisdictions/eu/gdpr_compliant.py, jurisdictions/index.py, tests/test_jurisdictions.py
  - Done when: jurisdiction_db.py exposes JurisdictionDB with methods: get(jurisdiction_key), list_all(), get_required_clauses(key), get_default_clause_values(key); california.py sets governing law to California/US, default term 2 years, CCPA-aware clauses; england_wales.py sets governing law to England & Wales, default term 3 years, GDPR-aware clauses; gdpr_compliant.py sets governing law to EU, default term 5 years, full GDPR compliance clauses; tests verify all 3 jurisdictions return correct governing law, default term, and required clauses (≥ 6 test cases)

- [ ] Task 4: Jinja2 template engine with variable substitution and clause composition
  - What: Build core/template_engine.py — a Jinja2-based engine that loads templates from the templates/ directory, performs variable substitution (party names, addresses, dates, etc.), composes clauses from the clause library, and renders the final NDA. Supports mutual, unilateral, and employment template types.
  - Files: core/template_engine.py, templates/mutual_nda.txt, templates/unilateral_nda.txt, templates/employment_nda.txt, tests/test_template_engine.py
  - Done when: template_engine.py exposes TemplateEngine class with methods: load_template(type), render(template_type, parties, jurisdiction, custom_clauses), get_available_templates(); mutual_nda.txt renders a mutual NDA with both disclosing/receiving party placeholders; unilateral_nda.txt renders a unilateral NDA with discloser/receiver placeholders; employment_nda.txt renders an employment NDA with employer/employee placeholders; all templates use Jinja2 variables for party details, jurisdiction info, and clause composition; tests verify template loading, variable substitution, clause composition, and output formatting (≥ 8 test cases)

- [ ] Task 5: CLI entry point with draft, customize, and validate subcommands
  - What: Build cli/main.py with argparse subcommands: draft (generate an NDA), customize (preview and modify clause values interactively), and validate (check an NDA for completeness). Support --template, --jurisdiction, --output, --clause (repeatable for overrides), and --interactive flags.
  - Files: cli/main.py, cli/commands/draft.py, cli/commands/customize.py, cli/commands/validate.py, cli/prompts.py, tests/test_cli.py
  - Done when: `python -m nda_contract_generator draft --template mutual --jurisdiction california --output output.txt` generates a complete NDA; `--clause term=3_years --clause remedies=injunctive` overrides specific clauses; `--interactive` mode guides user through party details and clause selection; `validate` subcommand checks for missing required fields and flags inconsistencies; all subcommands display help text; tests verify CLI argument parsing, subcommand routing, and output file creation (≥ 8 test cases)
