# Phase 3 Tasks

- [ ] Task 1: Design and implement plugin architecture
  - What: Build the plugin system that allows new device types and protocols to be added with < 100 lines of code. This includes a plugin loader, base plugin classes, entry-point discovery, and a plugin registry.
  - Files:
    - `src/pairenv/plugins/__init__.py` — Plugin base classes and registry
    - `src/pairenv/plugins/loader.py` — Dynamic plugin discovery and loading (entry points + filesystem)
    - `src/pairenv/plugins/base.py` — `PluginABC` abstract base class with `connect`, `disconnect`, `send`, `receive`, `discover`, `get_capabilities`
    - `src/pairenv/plugins/registry.py` — `PluginRegistry` to manage loaded plugins, versioning, and lifecycle
    - `src/pairenv/transports/mqtt_transport.py` — Example MQTT transport plugin
    - `src/pairenv/transports/wifi_transport.py` — Example WiFi/HTTP transport plugin
    - `src/pairenv/transports/ble_transport.py` — Example BLE transport plugin
    - `src/pairenv/transports/simulation_transport.py` — Simulation transport for testing without hardware
    - `src/pairenv/config/default_registry.json` — Update with plugin metadata
    - `tests/test_plugins.py` — Tests for plugin loading, registration, lifecycle, and interface compliance
  - Done when:
    - A new transport plugin can be written in < 100 lines by subclassing `PluginABC`
    - Plugin loader discovers and loads plugins from both entry points and a plugins directory
    - `PluginRegistry` tracks plugin state (loaded, active, error) and supports hot-reload
    - 3 example plugins (MQTT, WiFi, BLE) are implemented and loadable
    - Simulation transport works for testing without hardware
    - All plugin tests pass with pytest

- [ ] Task 2: Build multi-device coordinator and enhance NLP for complex commands
  - What: Implement a coordinator that can orchestrate sequences and parallel execution across ≥10 devices, and enhance the English parser to handle multi-step, complex commands.
  - Files:
    - `src/pairenv/coordinator.py` — `DeviceCoordinator` class with `execute_sequence()`, `execute_parallel()`, `execute_group()`, state tracking, retry logic, and failure handling
    - `src/pairenv/nlp/enhanced_parser.py` — `EnhancedEnglishParser` that handles multi-step commands (e.g., "start the experiment" → sequence of 5 commands), conditional logic, and parameterized templates
    - `src/pairenv/nlp/patterns.py` — Complex command patterns and multi-device coordination templates
    - `src/pairenv/nlp/__init__.py` — Module init
    - `src/pairenv/parser.py` — Extend with `EnhancedEnglishParser` alias and multi-step support
    - `src/pairenv/cli.py` — Add `execute` subcommand for multi-device sequences
    - `tests/test_coordinator.py` — Tests for sequence execution, parallel execution, ≥10 device sequences, retry logic, and failure recovery
    - `tests/test_enhanced_parser.py` — Tests for multi-step command parsing, complex templates, and parameterized commands
  - Done when:
    - Multi-device coordinator handles ≥10 device sequences reliably with retry and rollback
    - Enhanced parser handles ≥15 complex multi-step templates (e.g., "start the experiment", "calibrate all sensors", "run the full test suite")
    - Parallel execution across devices works correctly
    - All coordinator and parser tests pass with pytest

    - `src/pairenv/dashboard/routes/pairing.py` — Pairing management routes (add, remove, list, update)
    - `src/pairenv/dashboard/routes/devices.py` — Device monitoring routes (status, health, metrics)
    - `src/pairenv/dashboard/routes/commands.py` — Command issuance routes (single device, multi-device sequences)
    - `src/pairenv/dashboard/routes/plugins.py` — Plugin management routes (list, install, enable, disable)
    - `src/pairenv/dashboard/routes/api.py` — REST API endpoints for SDK integration
    - `src/pairenv/dashboard/templates/` — HTML templates (base, index, pairing, devices, commands, plugins)
    - `src/pairenv/dashboard/static/css/` — Stylesheets
    - `src/pairenv/dashboard/static/js/` — JavaScript (HTMX scripts, WebSocket client for live monitoring)
    - `src/pairenv/dashboard/config.py` — Dashboard configuration
    - `tests/test_dashboard.py` — Tests for dashboard routes, API endpoints, and template rendering
  - Done when:
    - Dashboard supports pairing management (add, remove, list, update device pairings)
    - Dashboard supports device monitoring (live status, connection state, metrics)
    - Dashboard supports command issuance (single device and multi-device sequences)
    - Dashboard supports plugin management (list, install, enable, disable)
    - REST API endpoints work for SDK integration
    - All dashboard tests pass with pytest

    - `src/pairenv/dashboard/config.py` — Add auth configuration
    - `tests/test_auth.py` — Tests for JWT auth, RBAC enforcement, pairing tokens, and permission scoping
    - `tests/test_security.py` — Security tests (token expiration, invalid token handling, permission denial)
  - Done when:
    - JWT authentication works for all dashboard and API routes
    - RBAC enforces permissions correctly for all 4 roles

    - Scoped command permissions prevent unauthorized actions
    - All auth and security tests pass with pytest
