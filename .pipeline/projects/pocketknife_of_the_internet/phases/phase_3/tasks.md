# Phase 3 Tasks

- [ ] Task 1: Taskbar Core — Persistent window list with previews
  - What: Build a Taskbar class that tracks all open windows and displays their current state (title, minimized/maximized/active status). The taskbar should render a visual preview for each window showing its title, icon/state indicator, and active highlight.
  - Files:
    - `/workspace/idea impl/.pipeline/projects/pocketknife_of_the_internet/workspace/core/taskbar/taskbar.py` (new — Taskbar class)
    - `/workspace/idea impl/.pipeline/projects/pocketknife_of_the_internet/workspace/core/taskbar/__init__.py` (new — package init)
  - Done when:
    - Taskbar class has properties: `windows` (list of tracked window refs), `position` (top/bottom/left/right), `height` (in pixels), `is_visible` (bool)
    - Taskbar has method `add_window(window_id, title, url, state)` that registers a window for tracking
    - Taskbar has method `remove_window(window_id)` that unregisters a window
    - Taskbar has method `update_window_state(window_id, state_updates)` that updates the tracked state of a window (e.g., minimized, active)
    - Taskbar has method `get_window_list()` that returns a list of dicts with window_id, title, url, is_minimized, is_maximized, is_active
    - Taskbar has method `get_active_window()` that returns the currently active window's info
    - Taskbar has method `render_preview(window_id)` that returns a dict with preview data: title, url, is_minimized, is_maximized, is_active, z_index
    - Taskbar can be initialized with position (default bottom) and height (default 48)
    - Taskbar remains visible and functional regardless of window state

- [ ] Task 3: Alt+Tab Window Switcher
  - What: Implement a window switching mechanism that cycles through open windows (like Alt+Tab on desktop). This includes tracking the switch order, showing a preview overlay of the currently selected window, and focusing the selected window when the switch completes.
  - Files:
    - `/workspace/idea impl/.pipeline/projects/pocketknife_of_the_internet/workspace/core/taskbar/window_switcher.py` (new — WindowSwitcher class)
    - `/workspace/idea impl/.pipeline/projects/pocketknife_of_the_internet/workspace/core/window_manager/window_manager.py` (modify — add switch_window method)
  - Done when:
    - WindowSwitcher class has properties: `is_active` (bool), `current_index` (int), `window_order` (list of window IDs in switch order), `preview_window_id` (int or None)
    - WindowSwitcher has method `activate()` that starts the switch cycle (sets is_active=True, shows preview of first window)
    - WindowSwitcher has method `deactivate()` that ends the switch cycle (focuses the preview window, resets state)
    - WindowSwitcher has method `next_window()` that advances to the next window in the cycle
    - WindowSwitcher has method `prev_window()` that goes to the previous window in the cycle
    - WindowSwitcher has method `get_preview()` that returns the current preview window's info dict
    - WindowSwitcher has method `focus_selected()` that focuses the current preview window and deactivates the switcher
    - WindowSwitcher has method `update_window_order(window_manager)` that rebuilds the switch order from the window manager (active window first, then by z-index)
    - Switching a window via the switcher correctly focuses it and updates z-index
    - The switch order persists across multiple activate/deactivate cycles

- [ ] Task 4: System Tray — Notifications and Status Indicators
  - What: Build a SystemTray class that provides a status indicator area on the taskbar for notifications, clock/time display, and system status icons (e.g., volume, network, battery). Supports adding/removing notifications with auto-dismiss.
  - Files:
    - `/workspace/idea impl/.pipeline/projects/pocketknife_of_the_internet/workspace/core/taskbar/system_tray.py` (new — SystemTray class)
  - Done when:
    - SystemTray class has properties: `notifications` (list of notification dicts), `indicators` (list of indicator dicts), `clock_enabled` (bool)
    - Each notification dict has: `id`, `title`, `message`, `timestamp`, `priority` (low/medium/high), `dismissed` (bool)
    - Each indicator dict has: `id`, `icon` (string), `tooltip` (string), `active` (bool)
    - SystemTray has method `add_notification(title, message, priority="medium", duration_ms=5000)` that creates a notification with auto-dismiss after duration
    - SystemTray has method `dismiss_notification(notification_id)` that marks a notification as dismissed
    - SystemTray has method `remove_dismissed_notifications()` that removes all dismissed notifications
    - SystemTray has method `add_indicator(icon, tooltip)` that adds a status indicator
    - SystemTray has method `remove_indicator(indicator_id)` that removes a status indicator
    - SystemTray has method `toggle_clock()` that enables/disables the clock display
    - SystemTray has method `get_clock_time()` that returns the current time as a formatted string (HH:MM:SS)
    - SystemTray has method `get_active_notifications()` that returns non-dismissed notifications
    - SystemTray has method `get_indicators()` that returns all indicators
    - Notifications have a `dismissed_at` timestamp for tracking auto-dismiss timing
