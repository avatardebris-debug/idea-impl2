# Phase 6 Tasks

- [ ] Task 1: Implement Multi-Monitor Support
  - What: Build a Monitor class representing a display (id, name, x, y, width, height, is_primary) and a MonitorManager class that detects and manages available monitors. Integrate with Window and WindowManager so windows can be created on specific monitors, span across monitors, and are clamped to monitor bounds during drag. Add a get_monitor_at(x, y) method to MonitorManager and a get_monitor_for_window() method to Window.
  - Files: workspace/core/window_manager/monitor.py (new), workspace/core/window_manager/window_manager.py (modify), workspace/core/window_manager/window.py (modify)
  - Done when: MonitorManager can create at least 2 Monitor instances with different bounds. Window creation accepts a monitor_id parameter and places the window on that monitor. Window dragging clamps position to the current monitor's bounds. Window.get_monitor_for_window() returns the correct Monitor. Windows can span across monitors when width/height exceeds a single monitor's dimensions. At least 10 unit tests cover monitor detection, window placement, bounds clamping, and cross-monitor spanning.

- [ ] Task 2: Implement Window Snapping
  - What: Add a SnapManager class that defines snap zones (left-half, right-half, top-full, bottom-left-quarter, bottom-right-quarter, corners) and detects when a window is being dragged near zone boundaries. When a window enters a snap zone during a drag, it animates into the snapped position. Integrate with WindowManager to apply snap positions and with Window to support snapped state tracking. Add snap_preference to UserPreferences.
  - Files: workspace/core/window_manager/snap_manager.py (new), workspace/core/window_manager/window_manager.py (modify), workspace/core/window_manager/window.py (modify), workspace/core/file_system/user_preferences.py (modify)
  - Done when: Dragging a window to the left edge snaps it to the left half of the monitor. Dragging to the right edge snaps to the right half. Dragging to the top edge maximizes it. Dragging to corners snaps to quarter positions. Snap state is tracked on the Window (is_snapped, snap_zone). Snap can be toggled off via UserPreferences. At least 15 unit tests cover all snap zones, edge detection thresholds, snap animation triggers, and preference toggling.

- [ ] Task 3: Implement Window Tiling
  - What: Add a TileManager class that manages tiling layouts (horizontal split, vertical split, grid, spiral). Provide methods to tile selected windows in a layout pattern, auto-tile all windows on a monitor, and undo the last tiling operation. Windows are automatically resized and repositioned to fill available space without overlap. Integrate with WindowManager for window placement and with UserPreferences for tile layout persistence.
  - Files: workspace/core/window_manager/tile_manager.py (new), workspace/core/window_manager/window_manager.py (modify), workspace/core/file_system/user_preferences.py (modify)
  - Done when: TileManager can tile 2 windows in a horizontal split (50/50), vertical split (50/50), and grid (equal quadrants). Auto-tile places all windows on a monitor in a spiral layout. Undo restores windows to their pre-tile positions. Windows never overlap during tiling. Layout is saved to UserPreferences and can be restored. At least 15 unit tests cover all tiling layouts, undo, auto-tile, and preference persistence.

- [ ] Task 4: Implement Advanced Keyboard Shortcuts
  - What: Build a ShortcutManager class that registers, stores, and dispatches keyboard shortcuts. Support global shortcuts (Ctrl+Alt+Left/Right for snap, Ctrl+Alt+T for tile, Ctrl+` for window switcher) and per-application shortcuts (configurable per window type). Add a settings UI entry in UserPreferences for customizing shortcuts. Integrate with WindowSwitcher for Alt+Tab cycling and with SnapManager/TileManager for shortcut-triggered actions.
  - Files: workspace/core/window_manager/shortcut_manager.py (new), workspace/core/taskbar/window_switcher.py (modify), workspace/core/file_system/user_preferences.py (modify)
  - Done when: ShortcutManager can register, unregister, and look up shortcuts by key combination. Global shortcuts trigger snap, tile, and window switcher actions. Per-application shortcuts are stored and dispatched correctly. UserPreferences stores custom shortcut mappings and loads them on startup. Alt+Tab cycles through windows in z-index order with visual feedback. At least 15 unit tests cover shortcut registration, dispatch, per-app overrides, preference save/load, and Alt+Tab cycling.

- [ ] Task 5: Integration Tests and End-to-End Verification
  - What: Write integration tests that exercise the full Phase 6 feature set: create windows across multiple monitors, snap them, tile them, switch between them with keyboard shortcuts, verify state persistence, and confirm performance with many windows. Write an end-to-end test scenario that mimics a power user workflow: open 8 windows on 2 monitors, snap 4 to the left monitor and tile 4 on the right, switch windows with shortcuts, save state, reload, and verify everything is restored correctly.
  - Files: workspace/tests/test_phase6_integration.py (new), workspace/tests/test_integration.py (modify)
  - Done when: Integration test suite has at least 30 test cases covering multi-monitor workflows, snap+tile combinations, shortcut chains, state persistence across reloads, and performance benchmarks (10+ windows across 2 monitors with snapping and tiling completes in under 200ms per operation). End-to-end test verifies the full power user workflow. All tests pass with pytest.
