# Phase 3 Tasks

- [ ] Task 1: Kelly criterion risk engine
  - What: Implement the Kelly criterion budget allocation engine that limits exposure to ≤10% of total budget at all decision points. Core formula: f* = (bp - q) / b where b = odds, p = win probability, q = 1 - p. p is dynamically updated from agent confidence and historical win rate. Fractional Kelly (half-Kelly) used for safety. Engine must be callable from the RL agent's action pipeline and return a capped budget allocation per channel.
  - Files: `rl_dropshipping/src/risk/kelly_engine.py`, `rl_dropshipping/src/risk/__init__.py`, `tests/test_kelly_engine.py`
  - Done when: `KellyEngine.allocate(total_budget, win_rate, confidence)` returns a value ≤ 0.10 * total_budget for all valid inputs; fractional Kelly (half-Kelly) is applied by default; engine correctly handles edge cases (win_rate = 0, win_rate = 1, confidence = 0); unit tests cover all edge cases and the 10% cap is enforced; integration test verifies engine rejects allocations exceeding the cap

- [ ] Task 3: Shadow mode — parallel prediction with outcome comparison
  - What: Build a shadow mode system where the RL agent makes predictions (product selection, pricing, ad budget allocation) in parallel with no real actions executed. Predictions are logged with timestamps and compared against actual outcomes after a configurable delay (e.g., 24 hours for ad performance). Compute prediction error metrics (MAE, RMSE on profit/ROI) and track prediction accuracy over time. Shadow mode must be configurable to run alongside any existing agent (rule-based or RL) without interference.
  - Files: `rl_dropshipping/src/shadow/predictor.py`, `rl_dropshipping/src/shadow/comparator.py`, `rl_dropshipping/src/shadow/store.py`, `rl_dropshipping/src/shadow/__init__.py`, `tests/test_shadow_predictor.py`, `tests/test_shadow_comparator.py`
  - Done when: `ShadowPredictor.predict()` logs actions without executing them; `ShadowComparator.compute_error()` returns MAE and RMSE on profit/ROI predictions vs. actuals; prediction error stays <5% over a 2-week simulated shadow period with mock data; shadow mode can run alongside the RL agent without affecting real decisions; `pytest tests/test_shadow_*.py` all pass

- [ ] Task 4: Gradual rollout pipeline — shadow to real budget scaling
  - What: Implement a staged rollout system that transitions the agent from shadow mode to real-world execution. Start at 10% real budget / 90% shadow, then scale based on performance thresholds (e.g., increase real allocation by 10% every 3 days if prediction error <5% and profit > baseline). Must include a promotion criteria check, a rollback criteria check, and manual override capability. State machine tracks current rollout stage (shadow_only, 10pct_real, 25pct_real, 50pct_real, 75pct_real, full_real).
  - Files: `rl_dropshipping/src/rollout/stage_machine.py`, `rl_dropshipping/src/rollout/promotion_criteria.py`, `rl_dropshipping/src/rollout/rollback_criteria.py`, `rl_dropshipping/src/rollout/__init__.py`, `tests/test_stage_machine.py`, `tests/test_promotion_criteria.py`, `tests/test_rollback_criteria.py`
  - Done when: Stage machine correctly transitions through all 6 stages based on promotion/rollback criteria; promotion requires prediction error <5% over 3+ consecutive evaluation windows; rollback triggers on any single day exceeding loss threshold; manual override can force any stage change; full rollout reaches 100% real budget over 4+ weeks in simulation; `pytest tests/test_rollout_*.py` all pass

- [ ] Task 5: Calibration layer — sim-to-real metric mapping
  - What: Build a calibration layer that maps simulation metrics to real-world metrics and applies correction factors. The layer must learn correction factors during shadow mode by comparing sim-predicted outcomes vs. actual real-world outcomes. Support per-metric correction factors (profit delta, traffic delta, conversion rate delta, CPC delta). Output calibrated predictions that account for the sim-to-reality gap. Store correction factors persistently and allow manual adjustment.
  - Files: `rl_dropshipping/src/calibration/factor_store.py`, `rl_dropshipping/src/calibration/mapper.py`, `rl_dropshipping/src/calibration/trainer.py`, `rl_dropshipping/src/calibration/__init__.py`, `tests/test_calibration_mapper.py`, `tests/test_calibration_trainer.py`
  - Done when: `CalibrationMapper.calibrate(sim_metric)` returns a real-world-estimated metric using learned correction factors; correction factors are learned during shadow mode via linear regression on sim-vs-real pairs; calibration achieves ±15% accuracy on profit predictions after 2 weeks of shadow data; factors are persisted to disk and loadable across sessions; `pytest tests/test_calibration_*.py` all pass

- [ ] Task 6: Monitoring dashboard and kill-switch

Dashboard displays: real-time P&L, agent confidence scores, budget utilization per channel, prediction error trends, rollout stage, and anomaly detection alerts. Dashboard must be accessible via web UI and CLI. Kill-switch must be independently testable via fault injection.
  - Files: `rl_dropshipping/src/monitoring/dashboard.py`, `rl_dropshipping/src/monitoring/alerts.py`, `rl_dropshipping/src/monitoring/kill_switch.py`, `rl_dropshipping/src/monitoring/anomaly_detector.py`, `rl_dropshipping/src/monitoring/__init__.py`, `rl_dropshipping/monitoring_dashboard.html`, `tests/test_kill_switch.py`, `tests/test_anomaly_detector.py`, `tests/test_dashboard.py`
  - Done when: Dashboard renders real-time P&L, confidence, budget utilization, and rollout stage; kill-switch triggers on loss > 2x Kelly bet and executes rollback (logs event, halts real actions, sends alert); anomaly detector flags prediction error spikes (>2σ from rolling mean); fault injection test verifies kill-switch triggers correctly; `pytest tests/test_kill_switch.py` passes with fault injection; `pytest tests/test_anomaly_detector.py` passes; dashboard loads in browser without errors
