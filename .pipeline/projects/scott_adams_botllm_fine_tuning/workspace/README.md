# Scott Adams BotLLM Fine-Tuning Pipeline

A complete pipeline for scraping Scott Adams' writing corpus, analyzing his style, and preparing data for LLM fine-tuning.

## Project Structure

```
workspace/
├── scraper/                    # Data collection
│   ├── __init__.py
│   ├── main.py                 # Pipeline orchestrator
│   ├── scott_adams_blog.py     # Blog scraper (scottadamsslog.com)
│   ├── twitter_archives.py     # Twitter/X archive loader + synthetic generator
│   ├── book_excerpts.py        # Book excerpt loader + synthetic generator
│   └── cleaner.py              # Text cleaning and deduplication
├── analysis/                   # Style analysis
│   ├── __init__.py
│   ├── quantitative.py         # Statistical analysis (tokens, TTR, POS, sentiment, n-grams)
│   ├── qualitative.py          # Stylistic analysis (rhetorical devices, themes, patterns)
│   └── style_report.md         # Comprehensive style analysis report
├── prompts/                    # Prompt engineering
│   └── style_prompt_template.md  # Scott Adams style prompt template
├── corpus/                     # Data storage
│   ├── raw/                    # Raw scraped data
│   └── processed/              # Cleaned corpus (corpus.jsonl)
├── README.md                   # This file
└── requirements.txt            # Python dependencies
```

## Quick Start

### 1. Install Dependencies

```bash
pip install -r requirements.txt
```

### 2. Run the Pipeline

```bash
cd workspace
python scraper/main.py
```

This will:
1. Scrape blog posts from scottadamsslog.com
2. Load Twitter/X archives (or generate synthetic data)
3. Load book excerpts (or generate synthetic data)
4. Clean and deduplicate the corpus
5. Write the final corpus to `corpus/processed/corpus.jsonl`

### 3. Run Style Analysis

```bash
python analysis/quantitative.py
python analysis/qualitative.py
```

### 4. Review Style Report

Open `analysis/style_report.md` for a comprehensive analysis of Scott Adams' writing style.

### 5. Use the Prompt Template

See `prompts/style_prompt_template.md` for a ready-to-use prompt template that captures Adams' style.

## Corpus Schema

Each sample in `corpus.jsonl` follows this schema:

```json
{
    "id": "unique_identifier",
    "text": "The actual text content",
    "source_type": "blog | twitter | book",
    "source_url": "URL where the content was found",
    "date": "YYYY-MM-DD",
    "author": "Scott Adams",
    "raw_html": "Original HTML (optional)"
}
```

## Scraping Details

### Blog Scraper
- **Source**: scottadamsslog.com
- **Method**: HTTP requests + BeautifulSoup parsing
- **Content**: Blog post titles, dates, and body text
- **Rate Limiting**: 1-second delay between requests

### Twitter/X Archiver
- **Source**: Twitter/X data archive (downloaded from x.com/settings/your_account)
- **Method**: JSON parsing of archive files
- **Fallback**: Synthetic tweet generation if archives unavailable
- **Synthetic**: Uses Adams' known themes and patterns

### Book Excerpt Loader
- **Source**: Published Scott Adams books
- **Method**: Text extraction from available sources
- **Fallback**: Synthetic excerpt generation if unavailable
- **Synthetic**: Uses Adams' known themes and patterns

## Cleaning Pipeline

1. **HTML Removal**: Strip all HTML tags
2. **Boilerplate Removal**: Remove navigation, ads, comments
3. **Whitespace Normalization**: Collapse multiple spaces/newlines
4. **Deduplication**: N-gram hash fingerprinting (5-grams)
5. **Length Filtering**: Minimum 50 characters

## Style Analysis

### Quantitative Analysis
- **Token Statistics**: Mean, median, std dev, percentiles
- **Vocabulary Richness**: Type-token ratio, unique tokens, top words
- **POS Distribution**: Part-of-speech frequencies
- **Sentiment Analysis**: VADER polarity scores
- **N-gram Frequency**: Top 3-grams and 5-grams

### Qualitative Analysis
- **Rhetorical Devices**: Detection of 10 rhetorical patterns
- **Sentence Patterns**: Statement/question/exclamation/ellipsis/dash/colon
- **Thematic Classification**: 10 thematic categories
- **Paragraph Structure**: Length distribution and type classification
- **Stylistic Signature**: Overall stylistic metrics

## Fine-Tuning Recommendations

Based on the style analysis:

1. **Preserve**: Probability framing, direct address, short paragraphs, rhetorical questions, system vs. willpower framing
2. **Avoid**: Overly formal language, emotional appeals, absolute statements, long paragraphs, complex jargon
3. **Weight**: Probability-related terms higher during training
4. **Monitor**: Key stylistic markers during generation
