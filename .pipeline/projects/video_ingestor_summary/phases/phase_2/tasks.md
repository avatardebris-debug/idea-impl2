# Phase 2 Tasks

- [ ] Task 1: Text chunking and embedding module
  - What: Build a text chunking utility that splits transcript full_text and segments into overlapping chunks suitable for vector search. Generate sentence-level embeddings using a lightweight embedding model (e.g., sentence-transformers/all-MiniLM-L6-v2). Store chunk metadata (job_id, segment indices, timestamps) alongside embeddings.
  - Files: `src/video_ingestor/chunker.py` (new), `src/video_ingestor/embeddings.py` (new), `src/video_ingestor/config.py` (add EMBEDDING_MODEL and CHUNK_SIZE settings)
  - Done when: Given a list of TranscriptSegment objects, `chunker.chunk()` returns a list of Chunk objects each with text, start/end timestamps, and a 384-dim embedding vector; chunks overlap by 20% to preserve context across boundaries; `embeddings.generate(text)` returns a normalized embedding array; unit tests verify chunk boundaries and embedding dimensions

- [ ] Task 2: Vector index integration (ChromaDB)
  - What: Integrate ChromaDB as the vector store backend. Create a VectorStore class that manages per-job collections, supports upserting chunks with metadata, and performs similarity search with configurable top-k. Add migration path so existing Phase 1 jobs can be indexed on demand.
  - Files: `src/video_ingestor/vector_store.py` (new), `src/video_ingestor/config.py` (add VECTOR_DB_PATH and VECTOR_TOP_K settings), `src/video_ingestor/storage.py` (add `index_transcript(job_id)` method that triggers chunking + vector upsert), `requirements.txt` (add chromadb), `pyproject.toml` (add chromadb to dependencies)
  - Done when: `VectorStore.upsert(job_id, chunks)` stores chunks with their embeddings and metadata in a ChromaDB collection; `VectorStore.search(job_id, query_text, top_k=5)` returns the 5 most similar chunks with their text and timestamp metadata; existing jobs can be indexed via `storage.index_transcript(job_id)`; integration test verifies search returns relevant chunks for a known query

- [ ] Task 3: Model-agnostic LLM harness
  - What: Create an LLM harness that abstracts over LLM providers. Start with OpenAI as the primary provider (via `openai` Python SDK) with a fallback to a local model path. Define prompt templates for summarization and Q&A. Support configurable model selection via settings.
  - Files: `src/video_ingestor/llm_harness.py` (new), `src/video_ingestor/prompts.py` (new), `src/video_ingestor/config.py` (add LLM_PROVIDER, OPENAI_API_KEY, LLM_MODEL settings), `requirements.txt` (add openai), `pyproject.toml` (add openai to dependencies)
  - Done when: `llm_harness.generate(prompt, model="gpt-4o-mini")` returns a text response from OpenAI; `prompts.SUMMARIZATION_PROMPT` and `prompts.QA_PROMPT` templates produce well-formed prompts with transcript context injected; LLM harness raises a clear error when the configured provider is unavailable; unit tests mock the API call and verify prompt construction

- [ ] Task 4: Summarization service and API endpoint
  - What: Build a summarization service that takes a completed job's transcript and produces a structured summary (short summary + key points + action items). Support configuration options for summary length (short/medium/long), tone (neutral/professional/casual), and format (paragraph/bulleted). Trigger summarization automatically after transcription completes, and expose a manual summarization endpoint.
  - Files: `src/video_ingestor/summarizer.py` (new), `src/video_ingestor/api.py` (add POST /summarize/{job_id} endpoint and auto-summarize hook), `src/video_ingestor/models.py` (add SummaryResponse model), `src/video_ingestor/storage.py` (add summary_json column to jobs table), `tests/test_summarizer.py` (new)
  - Done when: POST /summarize/{job_id} returns a JSON response with summary_text, key_points (list), and action_items (list); auto-summarization runs within 30 seconds of transcription completion; summary configuration via query params (e.g., ?length=long&tone=professional) affects output; unit tests verify summary structure and that different length/tone configs produce meaningfully different outputs

- [ ] Task 5: Q&A engine with RAG and citation support
  - What: Build the Q&A engine that accepts a user question for a given job, retrieves relevant transcript chunks via vector search, constructs a RAG prompt with the retrieved context, and generates a grounded answer. Answers must include citation references with accurate timestamps (±5 seconds) pointing to the source transcript segments.
  - Files: `src/video_ingestor/qa_engine.py` (new), `src/video_ingestor/api.py` (add POST /qa/{job_id} endpoint), `src/video_ingestor/models.py` (add QARequest and QAResponse models), `tests/test_qa_engine.py` (new)
  - Done when: POST /qa/{job_id} with body {"question": "..."} returns a JSON response with answer text, citations (list of {text, start, end}), and confidence score; citations reference actual transcript segments with timestamps within ±5 seconds of the cited content; Q&A latency is under 10 seconds for videos up to 30 minutes; integration test verifies that asking about a specific topic in a known transcript returns an answer grounded in that content with correct citations

- [ ] Task 6: API documentation, config updates, and integration tests
  - What: Update API documentation (OpenAPI/Swagger) with all new endpoints. Add comprehensive integration tests covering the full Phase 2 pipeline: ingest → transcribe → auto-summarize → index → Q&A. Update CLI to support summarization and Q&A commands. Ensure all new dependencies are declared and the project installs cleanly.
  - Files: `src/video_ingestor/api.py` (update docstrings, OpenAPI tags), `src/video_ingestor/cli.py` (add summarize and qa subcommands), `tests/test_integration.py` (add Phase 2 integration test), `requirements.txt` (verify all dependencies), `pyproject.toml` (verify all dependencies), `workspace/README.md` (update with Phase 2 API docs)
  - Done when: `pip install -e .[dev]` installs all Phase 2 dependencies without errors; `pytest tests/` passes including new Phase 2 integration tests; `uvicorn` starts with all new endpoints documented in Swagger UI; CLI supports `video-ingestor summarize <job_id>` and `video-ingestor qa <job_id> --question "..."`; README documents all Phase 2 endpoints with example curl commands
