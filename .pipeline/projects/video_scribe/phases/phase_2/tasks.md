# Phase 2 Tasks

- [x] Task 1: Scene segmentation module
  - What: Build a scene_segmenter module that detects scene boundaries in a video using frame-difference thresholding and color histogram comparison. Configurable sensitivity (default 15% pixel change) with minimum scene duration of 2 seconds to avoid false positives.
  - Files: video_scribe/scene_segmenter.py (new), video_scribe/config.py (add scene detection config options)
  - Done when: Given any valid video file, the module returns a list of (start_frame, end_frame, start_time, end_time) tuples for each detected scene; configurable via a `threshold` parameter (default 15.0) and `min_scene_duration` parameter (default 2.0 seconds); frame-difference and color histogram methods produce consistent results on a test video with known scene cuts; handles videos with no scene changes (returns single scene covering entire video).

- [x] Task 2: Multi-frame extraction per scene
  - What: Extend frame extraction to pick 2-3 key frames per scene (at 25%, 50%, and 75% of each scene's duration) instead of a single mid-video frame. Reuse the scene_segmenter output.
  - Files: video_scribe/frame_extractor.py (add multi_frame extraction function), video_scribe/scene_segmenter.py (already created in Task 1)
  - Done when: Given a video and scene boundaries, the module returns a list of lists of PIL Images (one inner list per scene, each containing 2-3 frames); frames are evenly spaced at 25%, 50%, and 75% of each scene's duration; for scenes shorter than 1 second, falls back to a single frame at the scene midpoint; all frames are valid PIL Images that can be passed to the VLM analyzer.

- [x] Task 3: Multi-frame VLM analysis with parallel processing
  - What: Extend the VLM analyzer to accept up to 3 frames per scene in a single multimodal API call. Add concurrent.futures-based parallel processing to analyze multiple scenes simultaneously, reducing total runtime.
  - Files: video_scribe/vlm_analyzer.py (add analyze_frames method for multi-frame, add analyze_scenes_parallel function), video_scribe/config.py (add concurrency config options)
  - Done when: Given a list of 2-3 PIL Images, the module sends them as a single multimodal API call and returns a structured dict with per-frame analysis plus a synthesized summary; the analyze_scenes_parallel function processes multiple scenes concurrently using ThreadPoolExecutor (default max_workers=4); total processing time for 10 scenes is under 2x the time of a single scene (demonstrating parallelism benefit); API errors for individual scenes are handled gracefully without crashing the entire batch.

- [x] Task 4: Cross-scene context enrichment
  - What: After all scenes are analyzed, run a second LLM pass (text-only) to generate transition descriptions between consecutive scenes and a global video summary. Use prompt templates for multi-frame synthesis and transition detection.
  - Files: video_scribe/context_enricher.py (new)
  - Done when: Given a list of scene analysis dicts, the module returns an enriched list where each scene has an additional `transition_to_next` field (description of how the scene transitions to the next, or None for the last scene) and a `global_summary` field (2-3 sentence overview of the entire video); the transition descriptions are coherent and reference visual/audio continuity between scenes; handles edge cases (single-scene video, last scene with no transition).

- [x] Task 5: Structured output formatter (Markdown + JSON) with progress indicator
  - What: Build a new output formatter that assembles all scene data into a well-organized markdown document with scene headers, timestamps, per-scene sections (content, camera techniques, lighting, composition, mood), and transition descriptions. Also support JSON output with a structured schema. Add a progress indicator (stderr-based) during processing.
  - Files: video_scribe/output_formatter.py (add format_as_json and format_multi_scene_markdown functions), video_scribe/cli.py (add progress reporting, --format flag for json/md output)
  - Done when: format_multi_scene_markdown produces a markdown document with H2 headers per scene (e.g., "## Scene 1 — 00:00:00 to 00:00:15"), per-scene subsections for content, camera techniques, lighting/composition/mood, and transition to next scene, plus a global summary at the end; format_as_json produces valid JSON matching the schema: { "title": str, "total_scenes": int, "duration": float, "scenes": [ { "number": int, "start_time": float, "end_time": float, "content_summary": str, "visual_elements": list, "camera_techniques": list, "lighting_composition_mood": str, "transition_to_next": str or null } ], "global_summary": str }; CLI accepts --format md (default) or --format json; progress indicator prints to stderr showing "Processing scene X of Y ..." during VLM calls.

- [x] Task 6: CLI integration, new flags, and end-to-end smoke test
  - What: Update the CLI to wire all Phase 2 modules together. Add new flags: --threshold (scene detection sensitivity), --min-scene-duration, --max-workers (parallelism), --format (md/json). Update the main pipeline to orchestrate segmentation → multi-frame extraction → parallel VLM → context enrichment → output. Add a smoke test with a synthetic multi-scene video.
  - Files: video_scribe/cli.py (update run function and parser), tests/test_pipeline.py (add multi-scene smoke test), requirements.txt (add scenedetect if used, or confirm no new deps needed)
  - Done when: Running `python video_scribe.py input.mp4 --output description.md --format md` processes a multi-scene video end-to-end and writes a complete markdown document with all scene descriptions and transitions; running `--format json` produces valid JSON output; --threshold flag correctly adjusts scene detection sensitivity; smoke test generates a synthetic 3-scene video, processes it, and validates that the output contains exactly 3 scenes with correct timestamps; total processing time for a 5-minute test video is under 10 minutes (including API calls).
