Capability registry — phased plan
Goal: move from “agents can run_shell anything if they guess” to “the system knows what exists, when to use it, how to invoke it safely,” and eventually registers new tools it builds.

This extends what you already have (reusable_tools.md, shared_libs/, context_cache.json, truth.md, reviewer extraction) instead of replacing it.

Phase 0 — Foundations (prep, ~1–2 days)
Objective: Agree on schema, ownership, and refresh triggers before writing code.

Phase 3 — Routing policy (tuned selection, ~4–7 days)
Objective: System recommends which capability to use for a task — still optional for the agent to follow.

Work

pipeline/capability_router.py
Input: task text, project slug, domain hints
Output: ranked list [{id, slug, score, reason, example_invoke}]
Methods (layered):
Rules: keyword → domain (URDF → robotics; “pytest” → dev tools)
Requires graph: only suggest if dependencies satisfied on disk
LLM critic (optional): small call: “pick top 3 from this table”
Integrate into goal_decomposer
For each branch: suggested_capabilities: [...]
Prefer reuse branches when match score > threshold
Integrate into phase_planner
Add “Reuse” section in tasks.md when router finds hits
--hermes routing
If task is open-ended research → kind=hermes_skill rows rank higher
New artifact: .pipeline/goals/<goal_id>/capability_plan.yaml (optional)

Exit criteria

Router returns sensible top-3 for 10 hand-written test queries (robotics, movie, CSV, SEO)
Goal decomposition entries include reuse suggestions where projects exist
Documented false-positive handling (low score → “build new”)
Phase 4 — Safe invocation (controlled execution, ~5–8 days)
Objective: One new tool: invoke known entrypoints with guardrails — not raw shell roulette.

Work

New tools in tools.py (or pipeline/capability_tools.py)
list_capabilities(domain?, status=verified)
describe_capability(slug)
invoke_capability(slug, args_json, cwd?) → runs whitelisted entrypoint only
Guardrails
Entrypoint must exist in registry with status=verified
cwd resolved from template; no .. escape
Timeout, max output size, allowlist env vars
Log every invoke → update success_count / fail_count
Wire into executor only first (not validator/reviewer)
Deprecate path: merge reusable_tools.md generation from SQLite export; single source of truth
Exit criteria

Agent can run e.g. json_diff_tool or a shared_lib CLI via invoke_capability without hand-rolled shell
Failed invokes mark capability broken after N failures (health check report)
Security review: no arbitrary shell string from model — only registry rows
